Spam Email Detection

A Machine Learning project that implemented and evaluated the accuracy of Decision Trees, Random Forests, AdaBoost, Naive Bayes and Fully Connected Neural Networks to detect spam vs non-spam emails.

Team Project | Stack: Python, Tensorflow, sklearn | Repository: here




Motivation

This was my final project for my Machine Learning class. We wanted to implement and compare algorithms for a very real-world scenario - email spam detection. We had about a month complete this project so we used our ML knowledge over the semester to implement and analyze the results.

Project Stack and Explanation

For this project we used a dataset from UC Irvine that contained 60% non-spam/40% spam email split with different features such as if the email contained capitalizations in weird places, or unusual characters. We randomly chose data points to put into training and testing datasets. We decided to test this dataset on five different algorithms - Decision Trees, AdaBoost, Random Forests, Naive Bayes and Full-Connected Neural Networks. We used Python, Tensorflow and sklearn to implement this project. We used Tensorflow for the Neural Networks and sklearn for the other algorithms.

Lessons Learned

It was a great experience to be able to take what we learned from our Machine Learning class and apply it to a real world scenario. I learned that we as computer scientists usually have biases about which algorithms will do better in certain scenarios but we must test many algorithms. To our surprise, AdaBoost performed the most accurately with spam classification.

In the future it would be interesting to apply this algorithm to an email service to see Machine Learning in action with web development.
